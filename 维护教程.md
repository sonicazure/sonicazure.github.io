## 一、维护的基本逻辑

### 核心原则

这个网站的所有**内容**都存储在 `src/data/` 文件夹下的数据文件中。你不需要改任何页面代码，只需要改数据文件，网站就会自动显示新内容。

### 文件对应关系

| 你要改的内容 | 编辑这个文件 |
|-------------|------------|
| 名字、学术经历、工作经历 | `src/data/profile.ts` |
| 博文 | `src/data/blog.ts` |
| 年度最推荐书籍 | `src/data/annualBooks.ts` |
| 每年阅读的所有书 | `src/data/readingHistory.ts` |
| 灵韵之书（文学/哲学/社会理论） | `src/data/influential.ts` |
| 音乐（年度专辑/私藏Live/共振之音） | `src/data/music.ts` |
| 研究项目 | `src/data/research.ts` |

### 编辑方式

有两种方式编辑：

**方式A：GitHub网页编辑（推荐）**
- 直接在浏览器里改
- 改完自动部署
- 适合小修改

**方式B：本地下载编辑后上传**
- 把整个项目下载到电脑
- 用文本编辑器改
- 适合大批量修改

下面教程以 **方式A（GitHub网页编辑）** 为主。

## 二、修改个人简介

### 打开文件

1. 打开你的 GitHub 仓库网页：`https://github.com/neuroamer/neuroamer.github.io`
2. 点击 `src` 文件夹 → `data` 文件夹 → `profile.ts`
3. 点击右上角的 ✏️ 铅笔图标进入编辑模式

### 修改名字

找到这一行：

```typescript
name: "豆鸣青",
nameEn: "Dou Mingqing",
```

改成你想要的名字。

### 修改学术经历

找到 `academicTraining` 部分：

```typescript
export const academicTraining = [
  { flags: "🇨🇳", degree: "**社会学 (法学学士)**", note: "*美好的走街串巷时光*" },
  // ...
];
```

格式规则：
- `flags`：国旗 emoji，直接复制粘贴，如 `🇨🇳`、`🇬🇧`、`🇧🇪`、`🇺🇸`
- `degree`：学位名称，用 `**`** 包裹表示**加粗**
- `note`：备注，用 `*` `*` 包裹表示*斜体*

**添加新行**：复制一行，改内容。注意每行末尾有逗号，最后一行没有逗号。

### 修改工作经历

**白领工作**：找到 `whiteCollarWorks`：

```typescript
export const whiteCollarWorks = [
  { flags: "🇨🇳", title: "**用户研究员**" },
  { flags: "🇺🇸", title: "**Research Associate**" },
];
```

**黑领工作**：找到 `blueCollarWorks`：

```typescript
export const blueCollarWorks = [
  { flags: "🇨🇳🇬🇧", title: "**交通局越帮越忙后备调度猿**" },
  // ...
];
```

**添加新工作**：复制一行改内容即可。

### 保存

1. 页面底部找到 **Commit changes** 区域
2. 第一个框写更新说明，如：`更新学术经历和工作经历`
3. 点击 **Commit changes**
4. 等 1-2 分钟，刷新网站 `https://neuroamer.github.io`

## 三、添加/修改博文

### 打开文件

1. 仓库网页 → `src` → `data` → `blog.ts`
2. 点击 ✏️ 铅笔图标

### 理解博文结构

```typescript
{
  id: "digital-labor-platform",     // 唯一英文标识（不能重复）
  title: "平台资本主义下的数字劳动",  // 文章标题
  date: "2024-03-15",                // 发布日期
  year: 2024,                        // 年份（用于筛选）
  tags: ["平台资本主义", "数字劳动", "媒介理论"],  // 标签
  excerpt: "当算法成为无形的管理者...",             // 列表页显示的摘要
  content: `                          // 正文开始（用反引号包裹）
## 第一节标题

这里是正文内容，可以写很长很长。

- 列表项1
- 列表项2

## 第二节标题

更多正文内容...

## 第三节

正文支持：
- 普通段落
- ## 二级标题（两个#号）
- - 列表项（减号开头）
- 空行（换行）
  `,                                 // 正文结束
  readingTime: 18                   // 预计阅读时间（分钟）
}
```

### 添加新博文

1. **找到最后一个博文条目**，在它后面添加新条目
2. 格式：复制一个现有的完整条目，改内容
3. **关键注意点**：
   - `id` 必须是唯一的英文标识，只能用字母、数字、横线，如 `"my-new-article"`
   - 新条目和上一个条目之间要有逗号隔开
   - 新条目如果是最后一个，末尾**不加逗号**

### 示例：添加一篇新博文

在 `blogPosts` 数组的最后（`}` 之前），添加：

```typescript
  {
    id: "my-new-article",
    title: "我的新文章标题",
    date: "2025-01-20",
    year: 2025,
    tags: ["社会理论", "基础设施"],
    excerpt: "这是一篇关于...的文章",
    content: `
## 引子

文章开头段落...

## 核心论点

主要内容...

- 要点一
- 要点二

## 结语

总结...
    `,
    readingTime: 12
  }
```

### 正文格式规则

| 效果 | 写法 | 示例 |
|------|------|------|
| 二级标题 | `## 标题` | `## 核心论点` |
| 普通段落 | 直接写 | `这是一段文字` |
| 列表项 | 减号开头 | `- 要点一` |
| 空行 | 空一行 | |

### 保存

1. 写提交说明：`添加新博文：我的新文章标题`
2. 点击 **Commit changes**
3. 等 1-2 分钟，刷新网站

## 四、管理书籍

### 年度书籍（最推荐的书）

文件：`src/data/annualBooks.ts`

每本书的结构：

```typescript
{
  id: "ab24-1",                    // 唯一标识
  title: "重构社会",                // 书名
  author: "Manuel Castells",       // 作者
  publisher: "社会科学文献出版社",  // 出版社
  year: 2023,                      // 出版年
  cover: "/images/book1.jpg",      // 封面图片路径
  doubanUrl: "https://book.douban.com/subject/35222540/",  // 豆瓣链接（可选）
  rating: 4.5,                     // 评分
  tags: ["网络社会学", "数字理论"], // 标签
  review: `书评内容...`             // 详细书评（可展开）
}
```

**添加新书**：找到对应年份的 `books` 数组，在末尾添加新条目。

**添加新年度**：在 `annualBestBooks` 数组末尾添加新年度块：

```typescript
  {
    year: 2025,
    books: [
      {
        id: "ab25-1",
        title: "新书标题",
        author: "作者名",
        publisher: "出版社",
        year: 2024,
        cover: "/images/book1.jpg",
        doubanUrl: "https://book.douban.com/subject/xxxxxx/",
        rating: 5,
        tags: ["标签1", "标签2"],
        review: `书评内容...`
      }
    ]
  }
```

### 阅读历史（每年读的所有书）

文件：`src/data/readingHistory.ts`

```typescript
{
  year: 2024,
  total: 23,                        // 总数
  navText: `这一年的阅读感想...`,    // 年度导航语（数百字）
  books: [
    { 
      id: "rh24-1", 
      title: "书名", 
      author: "作者", 
      cover: "/images/book1.jpg", 
      doubanUrl: "https://book.douban.com/subject/xxxxxx/",
      rating: 4.5, 
      tags: ["标签1"] 
    },
  ]

```

**添加新书**：在 `books` 数组末尾添加新条目。

**添加新年度**：复制一个年度块，改年份和内容。

### 灵韵之书（重要影响的作品）

文件：`src/data/influential.ts`

**添加新作者**：在对应分类的 `authors` 数组末尾添加：

```typescript
{ 
  name: "作者中文名", 
  nameEn: "Author English Name", 
  works: [
    { title: "《书名》", year: 2024, note: "备注内容" },
    { title: "《另一本书》", year: 2025, note: "" }
  ] 
},
```

**note** 是备注内容，展开后显示。如果暂时没有内容，写 `""`。

**同一作者添加新书**：在该作者的 `works` 数组中添加新条目。

## 六、管理研究项目

文件：`src/data/research.ts`

### 研究项目结构

```typescript
{
  id: "proj-id",                    // 唯一标识
  title: "项目中文标题",             // 标题
  subtitle: "英文副标题",            // 副标题（可选）
  status: "COMPLETED",               // 状态
  progress: 100                      // 进度百分比
}
```

### 状态说明

| 状态值 | 标签颜色 | 含义 |
|--------|---------|------|
| `COMPLETED` | 绿色 | 已完成 |
| `PENDING` | 黄色 | 待开展 |
| `IDEA` | 红色 | 构思中 |

### 在哪里添加

- **已完成的项目**：添加到 `completedProjects` 数组
- **待开展的项目**：添加到 `maturePlans` 数组
- **构思中的项目**：添加到 `futurePlans` 数组

## 七、添加图片

### 方式：GitHub网页上传

1. 打开仓库网页
2. 点击 `images` 文件夹
3. 点击 **Add file** → **Upload files**
4. 拖放图片文件到上传区域
5. 写提交说明：`添加书籍封面图片`
6. 点击 **Commit changes**

### 在数据文件中引用

图片上传后，路径格式为 `/images/文件名.jpg`，如：

```typescript
cover: "/images/my-new-book.jpg"
```

### 图片要求

- 格式：JPG 或 PNG
- 书籍封面：建议 2:3 比例（竖版）
- 音乐封面：建议 1:1 比例（正方形）
- 大小：每张不超过 500KB，太大的话网站加载慢

## 八、常见问题排查

### Q: 保存后网站没有更新？

1. 等 2-3 分钟（GitHub Pages 有缓存）
2. 检查仓库的 **Actions** 标签页，看是否有红色错误
3. 强制刷新：`Ctrl+Shift+R`（Windows）或 `Cmd+Shift+R`（Mac）

### Q: 页面显示空白或报错？

99%是因为格式错误。检查：

- [ ] 大括号 `{}` 是否成对匹配
- [ ] 每个条目末尾有逗号 `,`（**最后一个条目除外**）
- [ ] 反引号 `` ` `` 是否成对（正文开头和结尾）
- [ ] `id` 是否唯一（不能重复）
- [ ] 引号 `"` 是否成对

### Q: 怎么回退到之前的版本？

1. 在文件页面点击 **History**
2. 找到之前正常的版本
3. 点击右侧的 **Revert** 按钮

### Q: 我想一次性改很多内容，不想在网页上一点点改？

**方案：本地下载编辑**

1. 仓库网页 → 点击绿色 **Code** 按钮 → **Download ZIP**
2. 解压到桌面
3. 用任何文本编辑器（记事本、VS Code 等）打开 `src/data/` 下的文件
4. 批量修改
5. 修改完成后，在 GitHub 网页上逐个文件编辑，把修改后的内容粘贴进去

### Q: 我改错了但已经保存了，怎么恢复？

1. 打开那个文件
2. 点击 **History**
3. 找到改错之前的版本，点击
4. 点击 **Raw** 查看原始内容
5. 复制内容，回到最新版本粘贴恢复
